# План работ исправления ошибок

Дата: 2026-05-13. Issue: #23.

## Рекомендация

Начните с исправления и приемки Этапа 2: frontend MVP-прототип. Новые требования из issue #23 относятся к мобильным сценариям, визуальным состояниям и витринным скриншотам, поэтому их безопаснее закрывать до углубления backend/OCR/AI pipeline.

## Порядок работ

1. Этап 2: уточнить mobile UI и дизайн-систему.
   - Проверить экраны загрузки, отчета, истории, профиля и PWA-уведомлений.
   - Добавить недостающие состояния для оценки риска, чекапов, напоминаний и AI-чата.
   - Согласовать тексты, чтобы они не обещали постановку диагноза.

2. Этап 4: уточнить AI/OCR требования для новых экранов.
   - Зафиксировать JSON-поля для risk summary, checkups, medication reminders и chat agent actions.
   - Добавить правила медицинской безопасности для критических значений, лекарств и рекомендаций.

3. Этап 5: проверить безопасность и юридические ограничения.
   - Убедиться, что напоминания о лекарствах не выглядят как назначение лечения.
   - Проверить дисклеймеры для AI-ассистента, оценки риска и расшифровки анализов.

4. Этап 6: подготовить mobile/PWA приемку.
   - Проверить вертикальные viewport 591 x 1280 px и типовые размеры iOS/Android.
   - Проверить push-уведомления, offline shell и install flow.

5. Подготовка витрин.
   - Сгенерировать не менее 5 скриншотов на каждое из 5 заданий.
   - Отобрать финальные варианты для App Store и Google Play.
   - Перед публикацией провести юридическую и медицинскую редактуру текстов на изображениях.

## Почему начинать с Этапа 2

Этап 2 уже содержит текущий визуальный язык продукта и PWA-прототип. Если сначала уточнить интерфейс и скриншоты, команда быстрее увидит разрыв между текущим MVP и новыми ожиданиями, а backend/API задачи можно будет формализовать уже под подтвержденные экраны.
